Programmation web : HTML et CSS

Brève présentation du web

Le World Wide Web (littéralement la « la toile mondiale »), souvent abrégé en web, est un système fonctionnant sur internet permettant de consulter des documents (textes, musiques, vidéos). Ces documents, appelés pages web, sont regroupés sur des sites web qui sont accessible via les protocoles HTTP ou HTTPS à l'aide d'un navigateur web.

Quelques exemples de navigateur web.
  • Mozilla Firefox
  • Google Chrome
  • Safari
  • Microsoft Edge
  • Opera
  • Lynx

Le web a été inventé entre 1989 et 1990 par Tim Berners-Lee.

Les grandes étapes du développement web
  • 1980-1984 : Création du langage de balisage SGML. Il est basé sur le concept de balises et de liens hypertextes et propose pour la première fois de séparer la structure d'un document de sa présentation (imprimée, sur écran, etc.). Il est adopté par de nombreuses institutions et grandes entreprises comme format de stockage pour leur documentation technique (notamment le CERN).
  • 1986 : SGML devient la norme ISO 8879:1986.
  • 1991 : Tim Berners-Lee, informaticien britannique travaillant au CERN, annonce la première version du web. Ce dernier est basé sur un format de document inspiré du SGML mais très simplifié : HTML.
  • 1993 : Les premiers navigateurs web graphiques apparaissent. Le langage HTML évolue pour intégrer de nouvelles balises telles que les tables.
  • 1994-1997 création du W3C, organisme de normalisation qui publie la spécification du HTML 4.0 en 1997.
  • 2000-2007 : Le web se développe de manière rapide mais non concertée. HTML est étendu de manière non-standard par différents navigateurs de l'époque (Internet Explorer, Netscape, Mozilla Firefox).
  • 2007-2014 : Un long effort de modernisation et de standardisation du HTML est entrepris, ce qui donnera HTML 5 en 2014. Le format entre dans la modernité en standardisant l'intégration de contenus multimédia (audio et vidéo) et en facilitant l'adaptabilité des pages web (la page s'adapte automatiquement quel que soit le support : ordinateur, appareil mobile, page imprimmée, etc.).

Il est important de ne pas confondre web et internet : le web n'est qu'une des applications d'internet (même si c'est la plus visible). Par exemple, l'envoi de mails (protocole SMTP) ou leur réception (protocoles IMAP ou POP) se font grâce à internet mais pas sur le web.

Description d'une page web

La description de pages web s'effectue à l'aide de trois types de documents.

  • Le contenu d'une page web et sa structure sont décrites à l'aide de documents HTML.
  • L'apparence d'une page web est décrite à l'aide de feuilles de style CSS.
  • Les aspects interactifs d'une page sont décrites à l'aide d'un script JavaScript.

Aujourd'hui, nous allons voir comment utiliser le HTML et le CSS. Le JavaScript sera étudié plus tard dans l'année.

Comment écrire une page web ?

HTML

Le langage de balisage hypertexte ou HTML est un langage permettant de de décrire la structure et le contenu d'une page web. Il répond à la question « Qu'est-ce que dit la page ? ».

Le HTML est un langage de balisage. Ce n'est pas un langage de programmation ! L'objectif n'est pas d'effectuer des opérations (comme avec Python ou Scratch) mais de décrire un document.

Par exemple, regardons ce document HTML simple. Comme on peut le voir, un document HTML est constitué d'éléments, dont la structure est décrite ci-dessous.

Ces éléments sont imbriqués les uns dans les autres afin de former une arborescence (comme pour les widgets de TkInter). Certains éléments disposent d'attributs qui sont indiquées dans la balise ouvrante.

Les documents HTML ont toujours la même structure : celle du fichier structure.html.

Certaines balises (comme la balise meta) sont appellées des balises vides. Pour ces balises, on ne dispose pas de balise fermante : la balise est ouverte et fermée en même temps. Ce comportement est parfois signalé en écrivant <meta charset="utf-8" />.

Le HTML est un langage standardisé. Il est donc important de s'assurer que le document HTML est correct. Pour cela, on pourra utiliser le validateur de HTML du W3C.

Les balises

Les balises HTML sont regroupées en deux catégories : les balises de structure qui indiquent la structure du document, et les balises de texte dont l'effet porte sur un morceau de texte.

Quelques balises importantes

Pour savoir ce que font les balises et quand les utiliser, on se réfèrera à la documentation en ligne de la fondation Mozilla.

Les balises <h1>, <h2>, <h3>, <h4>, <h5> et <h6> permettent de définir des titres de sections, sous-sections, etc. jusqu'à six niveaux.

La balise <p> représente un paragraphe. Les espaces et les retours à la ligne sont ignorés.

La balise <ol> permet de créer une liste à numéros (ordrerd list). Chaque élément de la liste est identifié par un la balise <li>. On peut de la même manière créer une liste à puces (unordered list) à l'aide de la balise <ul>. Voici un exemple :

  • Une chose
  • Puis une autre
  1. Le premier point
  2. Le second point
    • Un sous-point du second point
    • Encore autre chose
Code HTML 
<ul>
  <li>Une chose</li>
  <li>Puis une autre</li>
</ul>
<ol>
  <li>Le premier point</li>
  <li>
    Le second point
    <ul>
      <li>Un sous-point du second point</li>
      <li>Encore autre chose</li>
    </ul>
  </li>
</ol>

On peut organiser du texte dans des tableaux à deux dimensions à l'aide de l'élément table. Les rangées de la table sont délimitées par l'élément tr (table row) et les cellules par td (table data). Voici un exemple :

1 2 3
A a b c d e f g h i j k l m n o p q r s t u v w x y z
B
Code HTML 
<table>
  <tr>
    <td>1</td> <td>2</td> <td>3</td>
  </tr>
  <tr>
    <td>A</td>
    <td rowspan="2" colspan="2">a b c d e f g h i j k l m n o p q r s t u v w x y z</td>
  </tr>
  <tr><td>B</td></tr>
</table>

L'élément a permet de mettre en œuvre des liens hypertextes, avec la syntaxe <a href="lien">texte du lien</a>.

Des images peuvent être insérée à l'aide la balise vide <img>, avec la syntaxe <img src="emplacement de l'image" alt="description rapide">. D'autres balises du même type existent pour insérer des fichiers multimédia.

Enfin, certaines balises n'ont a priori pas d'effet mais permettent d'indiquer le rôle de l'élément HTML : main pour le contenu principal, header pour l'en-tête de la page, footer pour son pied de page, article, section, div, span, etc. Ces balises peuvent ensuite servir à mettre en page le contenu selon son rôle à l'aide du CSS. Pour mettre en avant un petit morceau de texte, on pourra utiliser les balises em, strong, mark ou encore code.

Balises neutres, classes et identifiants

Il existe en HTML deux balises neutres (qui n'ont aucun effet ni aucune signification) : la balise de structure div (qui se comporte comme p) et la balise de texte span. On verra l'intérêt de ces balises dans la section suivante.

Pour chaque balise, il est possible d'ajouter deux attributs id et class. L'attribut id doit être unique dans tous les document. L'intérêt de ces attributs sera explicité dans la section suivante.

Les bases du CSS

Les feuilles de style en cascade ou CSS sont un moyen de décrire l'apparence d'une page web. Elles répondent à la question « Qu'est-ce que montre la page ? ».

Les CSS permettent de définir les propriétés graphiques des éléments HTML constituant une page web.

On écrit un paragraphe de texte, mais cette fois on modifie le style graphique des éléments.

Code HTML + CSS 
<p style="text-align:right;">
On écrit un
<span style="border:2pt solid black;"> paragraphe </span>de
texte, mais cette fois on
<span style="color:#1FE6B1;"> modifie </span> le
<span style="color:red; background:yellow">style</span>
graphique des
<span style="border:2pt solid black">éléments</span>.</p>

On remarque que l'attribut style permet de définir les propriétés CSS de l'élément HTML. On voit aussi l'intérêt de span, qui permet de spécifier le contenu dont on souhaite modifier le style.

Il est à noter que la plupart des navigateurs appliquent des mises en page par défaut à certains éléments. Ainsi, un élément b sera généralement affiché en gras, i en italique et code avec une police à chasse fixe. Il faut bien garder à l'esprit que cette mise en page n'est qu'un choix des navigateurs : certains navigateurs peuvent décider de faire autrement (le standard n'impose rien à ce sujet) et les CSS peuvent changer la mise en page de ces éléments.

Il ne faut donc pas utiliser ces balises pour la forme qu'elles font avoir au texte, mais pour leur signification (respectivement un élément portant à l'attention, termes techniques ou en langue étrangère, et morceau de code informatique). Le HTML indique ce que dit la page, le CSS indique ce que montre la page.

Écrire du CSS

La méthode précédente fonctionne mais est peu pratique. D'une part, le code HTML est peu lisible car les aspects graphiques se mélangent à la sémantique de la page. Pour éviter cela, on peut mettre à part le code CSS, comme c'est le cas par exemple dans le fichier css-ex2.html

Cette méthode utilise les sélecteurs CSS afin de définir le style graphique.

  • p { text-align: right; } indique que tous les éléments de type p seront alignés à droite.
  • .bord { border: 2pt solid black; } indique que tous les éléments dont la classe (définie par l'attribut class) est bord seront encadré d'une bordure noire continue de deux points d'épaisseur.
  • #mot-modifie { color: #1FE6B1; } indique que l'élément dont l'id est mot-modifie sera colorié en #1FE6B1, soit une espèce de vert canard.

En pratique, dès que le CSS devient suffisament important, il est courant de l'écrire dans un fichier à part, par exemple css-ex2.css. Pour indiquer dans le document HTML quel style utiliser, on ajoutera dans l'en-tête du fichier HTML la ligne <link href="lien-vers-le-fichier.css" rel="stylesheet" type="text/css">. Un tel exemple d'organisation est disponible dans le fichier css-ex2.html.

Comme le HTML, le langage CSS est standardisé par le W3C. On s'assurera donc de la conformité des feuilles de style à l'aide du validateur de CSS du W3C .

Quelques propriétés du CSS

Modèle de boîte

Chaque élément HTML possède une boîte rectangulaire qui délimite son contenu. Lorsque le navigateur dessine un élément, il définit une boîte rectangulaire possèdant plusieurs zones distinctes.

  1. Le contenu (content), zone interne dans laquelle l'élément est dessiné.
  2. L'ajustement (padding), zone intermédiaire entre le contenu et la bordure.
  3. La bordure (border), qu'il est possible de rendre visible à l'aide du CSS.
  4. La marge (margin), qui est la zone entre la bordure et les autre éléments.

La taille des trois zones externes peut être modifié par les propriétés CSS du même nom : padding, border et margin.

Nous avons vu qu'il existe de types de balises en HTML : les balises de structure (p, ul, etc.) qui créent une boîte et force un retour à la ligne, et les balises textuelles (a, strong, etc.) qui placent leur contenu dans le flot du texte. Ce comportement est en fait dirigé par la propriété CSS display, qui peut prendre la valeur inline (boîte dans le corps du texte), block (boîte imposant un retour à la ligne) ou none (boîte masquée, invisible).

Longueurs et couleurs

De nombreuses propriétés CSS indiquent des longueurs (taille d'une bordure, d'un ajustement, d'une police). Leur valeur est toujours un nombre suivi d'une unité, les unités acceptées étant le pixel (px) ou le point (pt, qui est la même unité que celle utilisée pour déterminer la taille de la police dans de nombreux logiciels). On peut aussi donner la taille de manière relative par rapport à la taille de l'élément contenant, on indique pour cela un pourcentage (%).

Pour les couleurs, cela fonctionne de manière similaire au module turtle de Python. On utilise soit le nom en anglais (red, gray, blue, LightGreen, etc.), soit la notation hexadécimale avec #rrvvbb où rr, vv et bb représentent des valeurs sur deux chiffres entre 00 et ff (soit entre 0 et 255 en base 10). Ainsi, le noir s'écrit #000000 et le magenta #ff00ff.

Sélecteurs CSS

Le langage CSS définit une notion de sélecteur, c'est à dire un moyen de déterminer à quels éléments s'applique une propriété. La syntaxe est très complexe, nous n'en présentons qu'un extrait ici.

Les règles CSS ont la forme suivante :

é1 é2 ... én {
  prop1: val1;
  prop2: val2;
  ...
  propk: valk;
}

La partie é1 é2 ... én est le sélecteur CSS. Il est constitué d'un certain nombre d'étapes. L'étape é1 sélectionne tous les éléments qui vérifient la condition énoncée par é1. L'étape é2 sélectionne tous les éléments qui vérifient la condition énoncée par é2 et qui sont contenus dans des balises renvoyés par é1. On répète ce processus jusqu'à arriver au dernier pas.

Considèrons la règle CSS ci dessous :

div li a { font-weight: bold; }

Cette règle s'applique à tous les liens (a) qui sont à l'intérieur d'un élément (li) eux-mêmes à l'intérieur d'une élément (div).

Un étape peut prendre la forme suivante :

  • un nom de balise, comme a, h2 ou main. L'étapesélectionne alors tous les éléments du type concerné.
  • un nom de classe, préfixé par un point, comme .bord ou .exemple. L'étape sélectionne alors tous les éléments de la classe concernée.
  • un identifiant, préfixé par un croisillon, comme #mot-modifie, #mot-style ou #css. L'étape sélectionne l'unique élément dont l'identifiant est donné, si cet élément existe (rien n'est sélectionné sinon).
  • une combinaison d'une balise et d'un nom de classe, par exemple div.exemple. L'étape sélectionne alors tous les élément du type et de la classe concernés.
  • un combinaison d'une balise et d'un identifiant, par exemple span#mot-modifie. L'étape renvoie alors l'unique élément du type concerné dont l'identifiant est donné.
  • le symbole *. L'étape sélectionne alors toutes les balises.

Attention à ne pas confondre div.menu et div .menu !

Résolution de conflits

Le dernier aspects du CSS est sa méthode de résolution des conflits. Elle répond à la question « Que faire quand plusieurs règles s'appliquent au même élément ? ».

En cas de conflit entre différentes sources, c'est la source lue le plus récemment qui l'emporte.

Considérons maintenant le fichier conflit2.html. Quelle sera l'apparence du texte ?

En cas de conflit au sein d'une même source, la règle est la suivante. Pour chaque règle en conflit, on calcule le nombre i d'identifiants qui apparaissent dans la règle, le nombre c de classes qui apparaissent dans la règle et le nombre b de balises qui apparaissent dans la règle (* ne compte comme aucun des trois). Ensuite, compare les sélecteurs selon le triplet (i, c, b) : si un selecteur a plus d'identifants que les autres, alors il est choisi. Sinon, on prends les sélecteurs ex-aequo et on les compare selon c. Si on a encore des égalités, on compare enfin selon le nombre de balises. Enfin, s'il y a encore égalité, c'est la dernière règle (la plus basse dans le fichier) qui est choisie.

Dans l'exemple ci-dessus, les triplets sont respectivement (1, 0, 0), (0, 0, 2) et (0, 1, 1). C'est donc la première règle qui s'applique.
